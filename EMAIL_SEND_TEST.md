# メール送信機能テストガイド

## 現在の実装状況

### 実装済み
✅ Edge Function (send-email) - デプロイ済み、ACTIVE状態
✅ RESEND_API_KEY - 設定済み
✅ フロントエンドのメール送信ロジック
✅ データベーススキーマ (deliveries, delivery_recipients, email_logs)
✅ メールHTMLテンプレート

### 未実装
❌ ファイルのSupabase Storageへのアップロード機能
❌ ダウンロードページでの実際のファイル取得機能

## テスト手順

### 1. ログインする
アプリケーションにログインしてください。

### 2. 新規送信ページへ移動
「新規送信」ボタンをクリックします。

### 3. 宛先を設定 (ステップ1)
- **重要**: テスト用のメールアドレスを入力してください
- 推奨: 自分のメールアドレスを使用
- 例: your-email@example.com

### 4. ファイルを選択 (ステップ2)
- 任意のファイルをアップロード
- 注意: 現在、ファイルは実際にはアップロードされませんが、メタデータは保存されます

### 5. メッセージを作成 (ステップ3)
- 件名: テスト送信
- 本文: これはメール送信機能のテストです

### 6. 配信オプション (ステップ4)
- デフォルト設定のままでOK

### 7. プレビュー (ステップ5)
- メールの見た目を確認

### 8. 確認・送信 (ステップ6)
- 内容を確認して「送信する」をクリック

## 期待される結果

### 成功時
1. 「送信完了」画面が表示される
2. メール送信結果セクションに「1/1件 送信成功」と表示される
3. 受信者のメールアドレスに緑色のチェックマークが表示される
4. 指定したメールアドレスにメールが届く

### メール内容
- 件名: 入力した件名
- 差出人: SecureShare (または設定した送信者名)
- 本文: 入力したメッセージ
- ファイル一覧が表示される
- 各ファイルに「ダウンロード」ボタンがある
- 「まとめてダウンロード(ZIP)」ボタンがある

### 注意事項
⚠️ ダウンロードリンクをクリックすると、現時点ではファイルアップロード機能が未実装のため、ファイルのダウンロードは失敗します。
これは正常な動作です。メール送信自体が成功していれば、テストは成功です。

## トラブルシューティング

### メールが届かない場合
1. 迷惑メールフォルダを確認
2. Resend APIの送信制限を確認
3. email_logsテーブルでエラーメッセージを確認

### エラーメッセージが表示される場合
- "RESEND_API_KEY is not configured" → API キーの設定を確認
- "Unauthorized" → 再ログインを試す
- その他のエラー → ブラウザコンソールとエラー詳細を確認

## 次のステップ

メール送信が成功したら、次の機能を実装する必要があります：
1. Supabase Storageへのファイルアップロード
2. ダウンロードページでのファイル取得
3. ZIPファイルの生成とダウンロード
